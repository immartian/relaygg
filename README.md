# relaygg
 This project aims to verify a secure TLS proxy that enables access to blocked websites by modifying the SNI field in the TLS handshake and relaying the correct destination via an out-of-band (OOB) communication channel over QUIC. The proxy intercepts TLS connections, extracts the real SNI, replaces it with a camouflage SNI, and securely transmits the real SNI to a remote peer, which then performs the actual TLS handshake with the intended server. This allows users to bypass SNI-based censorship while maintaining a valid TLS session. The primary challenge is ensuring that all clients, including cURL, correctly send the intended SNI when connecting through the proxy.
